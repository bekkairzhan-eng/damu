# План разработки BI Damu — MVP

**Срок:** 6 недель (~1,5 месяца)
**Скоуп:** Foreman A / B / C / D, Site Manager

**Почему 6, а не 8 недель:**
Фронтенд-прототип готов (22 страницы), бизнес-логика задокументирована в TASKS.md, API-контракты видны из хардкода. Дизайн, UX-согласования и большая часть фронтенд-разработки уже позади — остаётся заменить моки на реальные API-вызовы.

---

## Команда

| Роль | Занятость |
|------|-----------|
| Frontend (прототип готов) | 100% |
| Backend 1 — ядро и интеграции | 100% |
| Backend 2 — план, аттестация, рейтинг | 100% |
| DevOps | 50% (первая неделя — 100%) |

---

## Неделя 1 — Инфраструктура + контракты

**Кто:** DevOps + обе бэкенд-команды

- Dev-окружение: PostgreSQL, Redis, Kafka, S3, Keycloak
- DB-схема из модели данных в TASKS.md
- CI/CD пайплайн (деплой по push)
- Keycloak: realm, роли, тестовые пользователи
- Зафиксировать контракт с UnityBPM: формат webhook, payload, тестовая среда ← **критично сделать на этой неделе**

**Выход:** все сервисы подняты, бэкенд стартует, фронт авторизуется через Keycloak

---

## Неделя 2 — Профиль + справочники + Admin-панель

**Кто:** Backend 1 + Frontend (параллельно)

- `GET /api/me` — профиль из HRMS
- CRUD навыков, должностей, требований, карьерного графа, весов рейтинга, пользователей
- Frontend: убирает хардкод Admin-панели, добавляет auth через Keycloak-токен

**Выход:** Admin-панель полностью живая, профиль реальный

---

## Неделя 3 — Карьерный граф + план + навыки

**Кто:** Backend 2 + Frontend (параллельно)

- `GET /api/career-graph?cluster=` — граф из БД
- `GET/POST /api/me/plans` — создание/смена цели
- `GET /api/me/skills` — мои навыки с уровнями
- `POST /api/me/skills/:id/request-approval` → триггер в UnityBPM
- Webhook от UnityBPM → `skill_approval_history`
- Frontend: CareerMap, MyPlans, MySkills на реальных данных

**Выход:** карьерная карта, план и подтверждение навыков работают end-to-end

---

## Неделя 4 — Аттестация + HR-панель

**Кто:** Backend 2 + Backend 1 + Frontend (параллельно)

- `POST /api/me/assessments/request` → задача в UnityBPM
- `GET /api/me/assessments` — история
- `POST /api/hr/assessments/:id/result` — HR вносит результат
- Уведомления при апруве / результате аттестации
- Frontend: Assessment и HR-панель на реальных данных

**Выход:** полный цикл аттестации работает

---

## Неделя 5 — Рейтинг + прод-окружение

**Кто:** Backend 1 + Backend 2 + DevOps

- Kafka-консьюмер событий из LMS (завершение курса)
- Ночной батч: считает 5 компонентов, пишет в `profile_rating`
- SuperApp-компонент — заглушка (фиксированное значение)
- Frontend: Dashboard показывает реальный рейтинг, error-состояния на всех экранах
- Prod-окружение: повторить инфру, smoke-тесты

**Выход:** рейтинг считается автоматически, прод поднят

---

## Неделя 6 — Стабилизация

**Кто:** все

- Баги по результатам ручного тестирования
- Аудит-лог действий admin / hr
- Пагинация в Admin-таблицах (пользователи, навыки)
- Нагрузочный тест (4000 сотрудников)
- Финальный деплой

**Выход:** MVP в проде

|-----|--------|

| ARIA / доступность | Внутренний инструмент, не критично сейчас |
| Мультиязычность UI | Только RU на старте |
| Онбординг-тур в CareerMap | Низкий приоритет |

---

## Главный риск

**UnityBPM** — даже при быстром доступе к API, webhook-интеграция требует тестовой среды и дебага форматов. Контракт нужно зафиксировать на **неделе 1**, иначе недели 3 и 4 сдвигаются и весь план рассыпается.
