# BI Damu — тест-кейсы для прототипа

Документ описывает тест-кейсы для текущего React-прототипа BI Damu (`app/`). Прототип полностью фронтендовый: данные захардкожены в коде или лежат в `localStorage` браузера, реального бэкенда/БПМ/HRMS нет. Тест-кейсы проверяют **фактическое поведение того, что уже кликается**, а не итоговую бизнес-логику из ТЗ (расхождения между прототипом и ТЗ вынесены отдельно в раздел 21).

## Как пользоваться документом

- **Запуск:** `cd app && npm run dev`, открыть `http://localhost:5173`.
- **Сброс состояния прототипа:** DevTools → Application → Local Storage → удалить домен, либо в консоли браузера `localStorage.clear()` и обновить страницу (F5). Каждый тест-кейс, если не указано иное, подразумевает **чистое состояние** (localStorage очищен) как стартовую точку.
- **Переключение роли демо-пользователя:** по умолчанию ключ `auth:roles` не задан, и код подставляет `['employee','hr','admin']` — у демо-пользователя сразу все роли. Чтобы проверить ограничение доступа, в консоли: `localStorage.setItem('auth:roles', JSON.stringify(['employee'])); location.reload()`.
- **Приоритеты:** Critical — ломает основной сценарий; High — заметный функциональный дефект; Medium — второстепенная функция/визуал; Low — косметика или задокументированная особенность прототипа.
- **Пометка ⚠️** — известное ограничение/дефект прототипа, обнаруженный при разборе кода. Не заводить повторно как «новый баг», если по итогам ретеста поведение не изменилось — сверяться с разделом 21.

## 1. Сквозные проверки (роли, тема, лэйаут, уведомления)

**Роли и доступ**

**TC-CROSS-001 — Доступ ко всем панелям по умолчанию [Medium]**
Шаги: очистить localStorage → открыть `/admin`, затем `/hr`.
Ожидаемо: обе панели открываются без ограничений (демо-роли включают admin и hr по умолчанию).

**TC-CROSS-002 — Блокировка `/admin` без роли admin [High]**
Шаги: `localStorage.setItem('auth:roles', JSON.stringify(['employee']))` → reload → открыть `/admin`.
Ожидаемо: экран «Нет доступа» (иконка замка), текст «У вас нет прав для просмотра этого раздела», кнопка «Вернуться на главную» → `/dashboard`.

**TC-CROSS-003 — Блокировка `/hr` без роли hr [High]**
Шаги: аналогично TC-CROSS-002, для `/hr`.
Ожидаемо: тот же экран «Нет доступа».

**TC-CROSS-004 — Переключатель панелей в TopNav [Medium]**
Шаги: с ролями `['employee','hr','admin']` открыть дропдаун кнопки «+2» в шапке.
Ожидаемо: пункты «HR-панель» (→ `/hr`) и «Панель администратора» (→ `/admin`), заголовок «Переключить панель». С одной доп. ролью кнопка показывает «HR» или «Admin» вместо «+2»; с ролью только `employee` кнопка-переключатель не отображается вовсе.

**TC-CROSS-005 — Сотруднические роуты не защищены [Low]**
Шаги: с ролью только `employee` открыть `/dashboard`, `/plans`, `/career-map`, `/titles`, `/skills`, `/assessment`.
Ожидаемо: все открываются без ограничений — `RoleGuard` применён только к `/admin/*` и `/hr/*`, ⚠️ это фиксируемая особенность прототипа, а не баг.

**Тёмная тема**

**TC-CROSS-006 — Переключение и персистентность темы [Medium]**
Шаги: нажать иконку темы в TopNav → обновить страницу.
Ожидаемо: вся страница инвертирует цвета (CSS-фильтр), выбор сохраняется после reload (`localStorage.theme:dark`), tooltip иконки меняется «Тёмная тема» ↔ «Светлая тема».

**TC-CROSS-007 — Логотип/аватар не инвертируются повторно [Low]**
Шаги: включить тёмную тему, посмотреть на логотип и аватар в шапке.
Ожидаемо: изображения выглядят нормально благодаря классу `invert-back` (двойная инверсия).

**Сайдбар и лэйаут**

**TC-CROSS-008 — Сворачивание сайдбара [Low]**
Шаги: нажать кнопку-шеврон на границе сайдбара.
Ожидаемо: ширина меняется 244px↔64px, подписи пунктов скрываются в свёрнутом виде, tooltip кнопки — «Развернуть»/«Свернуть».

**TC-CROSS-009 — Состояние сайдбара не сохраняется [Low] ⚠️**
Шаги: свернуть сайдбар → обновить страницу.
Ожидаемо: состояние сбрасывается к авторасчёту по ширине окна (не персистится).

**TC-CROSS-010 — Мобильная нижняя навигация [Medium]**
Шаги: уменьшить ширину окна/эмулировать мобильное разрешение.
Ожидаемо: боковой сайдбар и кнопка сворачивания скрыты, снизу — фиксированная панель из 5 пунктов (Главная, Развитие, Карьера, Должности, Навыки).

**Уведомления (колокольчик)**

**TC-CROSS-011 — Счётчик и список уведомлений [Medium]**
Шаги: открыть колокольчик.
Ожидаемо: бейдж = число непрочитанных из 5 демо-уведомлений; кнопка «Отметить все прочитанными» видна только пока есть непрочитанные.

**TC-CROSS-012 — Отметить все прочитанными [Medium]**
Шаги: нажать «Отметить все прочитанными».
Ожидаемо: бейдж пропадает, кнопка сама скрывается.

**TC-CROSS-013 — Чекбокс новостных уведомлений [Low]**
Шаги: снять чекбокс «Получать уведомления о новостях и предложениях» (включён по умолчанию).
Ожидаемо: уведомления типа news полностью пропадают из списка (не просто гаснут визуально).

**TC-CROSS-014 — Пустой список уведомлений [Low]**
Шаги: подобрать комбинацию (отметить все + снять чекбокс новостей), при которой видимых уведомлений не остаётся.
Ожидаемо: иконка `notifications_off` + текст «Нет уведомлений».

**TC-CROSS-015 — Декоративные элементы TopNav [Low] ⚠️**
Шаги: кликнуть кнопку «Apps», выбрать другой язык в переключателе, кликнуть аватар, кликнуть ссылку «Все →» над списком уведомлений.
Ожидаемо: язык меняет только отображаемый код (интерфейс не переводится, выбор не сохраняется между reload); остальные клики не дают эффекта — задокументированная особенность прототипа, не баг.

**Устойчивость к повреждённым данным**

**TC-CROSS-016 — Повреждённый JSON в localStorage [Medium]**
Шаги: `localStorage.setItem('careermap:goal', '{invalid json')` → открыть `/career-map`.
Ожидаемо: приложение не падает, тихо откатывается на значение по умолчанию `'Foreman A'`, ошибка нигде не показывается пользователю.

## 2. Моя страница (`/dashboard/my-dashboard`)

**TC-DASH-001 — Виджет рейтинга по должности [High]**
Шаги: открыть страницу.
Ожидаемо: по умолчанию виджет в режиме «по должности», кнопка подписана «Показать общий рейтинг», бейдж «#3 из 7», текущий пользователь подсвечен и подписан «Вы», топ-3 — медали 🥇🥈🥉.

**TC-DASH-002 — Переключение на общий рейтинг [Medium]**
Шаги: нажать «Показать общий рейтинг».
Ожидаемо: список меняется на общий (10 человек), кнопка меняет текст на «← Только Foreman B», бейдж места скрывается в этом режиме.

**TC-DASH-003 — Подтверждение навыка из списка ожидающих [High]**
Шаги: нажать «Подтвердить» у одного навыка из «Навыки, ожидающие подтверждения».
Ожидаемо: навык исчезает из списка ожидающих, появляется зелёный баннер «Подтверждено: {название} — добавлено в профиль навыков».

**TC-DASH-004 — Отклонение навыка [Medium]**
Шаги: нажать «Отклонить» у одного навыка.
Ожидаемо: навык исчезает из списка ожидающих; зелёный баннер подтверждения НЕ появляется (отклонённые навыки нигде не сохраняются).

**TC-DASH-005 — Обработка всех навыков только отклонением [Medium] ⚠️**
Шаги: отклонить все 8 навыков из списка ожидания, ни один не подтверждая.
Ожидаемо: список пуст, но текст «Все навыки обработаны» **не появляется** (условие показа требует хотя бы одного подтверждённого) — секция остаётся молча пустой без пояснения. Задокументировать как ограничение UX.

**TC-DASH-006 — Обработка всех навыков подтверждением [Medium]**
Шаги: подтвердить все 8 навыков по очереди.
Ожидаемо: после последнего — список ожидания пуст, зелёный баннер со всеми подтверждёнными названиями, текст «Все навыки обработаны» отображается.

**TC-DASH-007 — Переход по карточке плана [High]**
Шаги: нажать на карточку активного плана «Стать {цель}».
Ожидаемо: переход на `/plans` с автоматическим открытием карточки этого плана (передаётся `state.planId`).

**TC-DASH-008 — Карточка плана без данных [Low]**
Шаги: посмотреть на карточку «План развития на основе оценки».
Ожидаемо: вместо прогресс-бара — текст «Навыки не указаны. Получите персональные рекомендации на основе оценки!».

**TC-DASH-009 — Просроченный план подсвечен красным [Low]**
Шаги: посмотреть на карточку «Предыдущий карьерный план» (дедлайн «30 Авг 2024», в прошлом).
Ожидаемо: дедлайн отображается красным (плашка `expired`).

**TC-DASH-010 — Ссылка «Подробнее →» в блоке рейтинга [Low] ⚠️**
Шаги: нажать «Подробнее →» рядом с общим баллом.
Ожидаемо: переход на `/dashboard/experience`, но через **полную перезагрузку страницы** (обычная `<a href>`, а не SPA-навигация) — в отличие от всех остальных переходов на странице.

**TC-DASH-011 — Переходы из профиля [Low]**
Шаги: нажать чипы «Мои навыки» и «Ключевые компетенции» в карточке профиля.
Ожидаемо: переходят на `/dashboard/my-skills` и `/skills` соответственно (через SPA-навигацию).

## 3. Мои навыки (`/dashboard/my-skills`)

**TC-MYSK-001 — Категории по умолчанию [Medium]**
Шаги: открыть страницу без действий.
Ожидаемо: раскрыты только «Языки (CEFR)» и «Строительные практики», «Технологии» и «Управление и лидерство» свёрнуты.

**TC-MYSK-002 — Легенда уровней открыта по умолчанию [Low]**
Шаги: открыть страницу.
Ожидаемо: панель легенды видна сразу; кнопка «Легенда уровней» переключает видимость.

**TC-MYSK-003 — Фильтры (Все/Избранные/Подтверждённые/Целевые) [High]**
Шаги: последовательно кликнуть каждый фильтр.
Ожидаемо: активный фильтр подсвечивается сплошным синим; счётчик «{N} навыков» в заголовке каждой категории пересчитывается под активный фильтр; категория без совпадений скрывается целиком.

**TC-MYSK-004 — Глобальное пустое состояние [Medium]**
Шаги: выбрать фильтр, под который не попадает ни один навык (например, «Целевые», если ни один навык не помечен целевым в открытых категориях — подобрать сочетание).
Ожидаемо: иконка `search_off` + текст «Нет навыков по выбранному фильтру».

**TC-MYSK-005 — Избранное (звёздочка) [Low]**
Шаги: кликнуть звёздочку у навыка.
Ожидаемо: звёздочка становится жёлтой/закрашенной; повторный клик снимает избранное.

**TC-MYSK-006 — Родные языки без редактирования [Low]**
Шаги: посмотреть на строки «Казахский» и «Русский» в категории «Языки».
Ожидаемо: вместо уровня — зелёная плашка «Родной», без возможности редактирования.

**TC-MYSK-007 — Изменение уровня CEFR (Английский) [High]**
Шаги: у навыка «Английский» открыть выпадающий CEFR-бейдж, выбрать другой уровень (например, C1).
Ожидаемо: бейдж меняется на выбранный уровень; навык становится `selfDeclared` и теряет статус `confirmed` (если был подтверждён) — флаг «подтверждён» исчезает.

**TC-MYSK-008 — Изменение уровня по шкале 1–4 (не языковой навык) [High]**
Шаги: у любого навыка из «Строительные практики»/«Технологии»/«Управление» кликнуть другой сегмент шкалы уровня.
Ожидаемо: уровень меняется, наведение показывает подсказку («Начальный»/«Средний»/«Продвинутый»/«Эксперт»); навык становится `selfDeclared`, теряет `confirmed`.

**TC-MYSK-009 — Изменения не сохраняются между перезагрузками [Medium] ⚠️**
Шаги: изменить уровень/избранное у нескольких навыков → обновить страницу.
Ожидаемо: все изменения сброшены к дефолтным данным — экран не использует localStorage (только `useState`).

**TC-MYSK-010 — Декоративные элементы [Low] ⚠️**
Шаги: нажать «+ Добавить навык», «Только пробелы в навыках», «Автообновление навыков», «Выбрать», иконку info у любого навыка.
Ожидаемо: ни одно нажатие не даёт эффекта — все без обработчиков, задокументированная особенность прототипа.

## 4. Профиль опыта (`/dashboard/experience`)

**TC-EXP-001 — Базовый расчёт рейтинга профиля [High]**
Шаги: открыть страницу при чистых данных (2 проекта, 3/3 отмеченных пункта корп. активности, сертификат "Действителен" есть).
Ожидаемо: общий балл рассчитывается по формуле (навыки/обучение/SuperApp — фиксированные значения-заглушки, профиль и корп. активность — динамические) и отображается как `{X.X} / 5.0`. При первом заходе на страницу значение на «Моей странице» (виджет рейтинга) обновляется с дефолтных 4.3 на реально рассчитанное.

**TC-EXP-002 — Значение на Моей странице до первого визита в Профиль опыта [Medium] ⚠️**
Шаги: очистить localStorage, сразу открыть `/dashboard/my-dashboard` (не заходя на `/dashboard/experience`), посмотреть на общий балл.
Ожидаемо: показывается захардкоженное стартовое значение 4.3, а не реальный расчёт — балл обновляется только после первого монтирования компонента «Профиль опыта» за сессию.

**TC-EXP-003 — Добавление проекта: валидация названия [High]**
Шаги: открыть модалку «+ Добавить проект», оставить поле «Название объекта / проекта *» пустым, нажать «Сохранить».
Ожидаемо: ничего не сохраняется, модалка не закрывается (кнопка визуально приглушена, но фактически не задизейблена — просто no-op при пустом/пробельном названии).

**TC-EXP-004 — Добавление проекта: успешный сценарий [High]**
Шаги: заполнить название, выбрать тип объекта, указать роль/даты, нажать «Сохранить».
Ожидаемо: проект добавляется в список, модалка закрывается; при достижении 3+ проектов рейтинговый бонус проекта достигает максимума, подсказка «+X% за проект» скрывается.

**TC-EXP-005 — Удаление проекта без подтверждения [Medium] ⚠️**
Шаги: нажать иконку-корзину на карточке проекта.
Ожидаемо: проект удаляется немедленно, без диалога подтверждения.

**TC-EXP-006 — Пустое состояние «Мои проекты» [Low]**
Шаги: удалить все проекты.
Ожидаемо: текст «Добавьте проекты, в которых вы участвовали как член команды».

**TC-EXP-007 — Модалка «Добавить сертификат» не сохраняет [Critical] ⚠️**
Шаги: открыть «+ Добавить», заполнить название и остальные поля, нажать «Сохранить».
Ожидаемо (текущее поведение): модалка просто закрывается, новый сертификат **не появляется** в таблице — кнопки «Отмена» и «Сохранить» вызывают идентичный обработчик (закрыть модалку), реального сохранения нет. Явный дефект прототипа, стоит зафиксировать отдельным багом, а не просто как ограничение.

**TC-EXP-008 — Сортировка сертификатов не работает [Low] ⚠️**
Шаги: изменить значение в селекте «Сортировка: Дата выдачи/Срок истечения».
Ожидаемо: список не пересортировывается — у селекта нет обработчика.

**TC-EXP-009 — Заявка на обучение (истекающий сертификат) [High]**
Шаги: на баннере истекающего сертификата («Пожарная безопасность (уровень 2)») нажать «Записаться →», заполнить/оставить пустой желаемую дату, нажать «Отправить заявку».
Ожидаемо: заявка отправляется **без какой-либо валидации полей** (дату можно не указывать), показывается экран успеха «Заявка отправлена» с кнопкой «Закрыть».

**TC-EXP-010 — Корпоративная активность нередактируема [Medium] ⚠️**
Шаги: попытаться кликнуть по строкам блока «Корпоративная жизнь».
Ожидаемо: строки не реагируют на клик, несмотря на наличие в коде функции переключения (мёртвый код, не подключён к разметке) — итоговый балл всегда считается по дефолтным 3/3 отмеченным пунктам.

**TC-EXP-011 — Компонентные ссылки-скроллы [Low]**
Шаги: нажать «Мои программы →», «Рейтинг в SuperApp →», «Мои проекты →», «Корп. жизнь →».
Ожидаемо: страница плавно скроллит к соответствующему разделу тем же URL (`/dashboard/experience`), без смены роута. Только «Перейти к навыкам →» переходит на другую страницу (`/dashboard/my-skills`).

**TC-EXP-012 — Статус сертификата отображается корректно [Medium]**
Шаги: посмотреть на таблицу «Сертификаты».
Ожидаемо: зелёная плашка «Действителен» / красная «Истёк» в зависимости от даты; пустая дата истечения — «—».

## 13. Известные ограничения прототипа и расхождения с бизнес-спецификацией

| 5 | Статус «на скамейке» после успешной аттестации | При 100% пройденных навыков — статус «на скамейке» | В `HR/AssessmentEntry.jsx` такого статуса/перехода нет вовсе — сохраняется только `passed/total`, без какой-либо специальной пометки |
| 6 | Заявка на обучение (интеграция с LMS, статусы pending/matched/completed/rejected/withdrawn) | Полный флоу со сверкой раз в сутки | В прототипе есть только: (a) кнопка «Записаться» в Профиле опыта, которая ничего не сохраняет и всегда показывает «успех»; (b) HR-панель «Сертификаты» с частичной имитацией дублей — полноценных статусов заявки в коде нет |
| 7 | Матрица ответственных (кто по какому навыку/оргструктуре апрувит) | Настраивается в UnityBPM, гибко по навыку и оргструктуре | В прототипе HR всегда один и тот же человек — «Айгерим Сейткалиева», захардкожено во всех местах (`HR_FROM_MATRIX` и аналоги) |
| 8 | Аудит-лог всех действий | Отдельная сущность с полной историей изменений | В прототипе не реализовано вообще — ни одного экрана/хранилища аудита нет |
