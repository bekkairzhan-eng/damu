## 1. Роли в системе

| Роль | Кто | Что может |
|------|-----|-----------|
| **Сотрудник** | ИТР (Foreman, Site Manager и др.) | Смотрит свой профиль, подтверждает базовые/средние навыки сам, запрашивает аттестацию |
| **Руководитель** | Прямой руководитель сотрудника | Подтверждает продвинутые навыки подчинённых, видит профили своей команды, получает уведомления по команде |
| **Эксперт предметной области** | Специалист в конкретной области | Подтверждает навыки уровня «Эксперт» в своей зоне ответственности |
| **HR-партнёр** | HR, закреплённый за подразделением | Управляет матрицей навыков, собирает комиссию аттестации, исполняет задачи в БПМ |
| **Администратор** | Системный администратор | Управляет справочниками (должности, навыки, подразделения), назначает роли пользователям |

Авторизация — через Keycloak (SSO). Разработчик настраивает на старте.

## 2. Логика подтверждения навыков

> **Пересмотрено по итогам встречи с заказчиком 23.07.2026** (замечание Дильдар). Раньше уровни 1–2 сотрудник подтверждал сам без внешнего апрува — это оказалось неверным решением, отменено. Теперь сотрудник не подтверждает сам ни один уровень.

Каждый навык в матрице имеет 4 уровня: **Базовый → Средний → Продвинутый → Эксперт**.

Подтверждение теперь идёт разными источниками в зависимости от уровня, а не по единой метке «кто подтверждает»:

| Уровень | Кто/что подтверждает | Механизм |
|---------|------------------------|----------|
| 1 — Базовый | **BILIM** (LMS) | Автоматически, по результатам курса/теста внутри BILIM |
| 2 — Средний | **BILIM** (LMS) | Автоматически, аналогично уровню 1 |
| 3 — Продвинутый | **TestLab** (новая система тестирования) | Автоматически, по результатам тестирования в TestLab |
| 4 — Эксперт | Эксперт / непосредственный руководитель | Вручную, единственный уровень с ручным апрувом через UnityBPM |

Точный триггер подтверждения в BILIM (завершение курса? проходной балл теста?) и порог зачёта в TestLab — не определены, требуют отдельной проработки с командами этих систем перед реализацией интеграции.

### Статусы навыка после действия сотрудника:

| Статус | Когда | Отображение |
|--------|-------|-------------|
| ⏳ **Ожидает подтверждения** | Уровень задан, ждём ответа от BILIM / TestLab / эксперта | Жёлтый, в прогресс не засчитывается до подтверждения |
| ✅ **Подтверждён** | Пришёл положительный результат от BILIM / TestLab / эксперта | Зелёный, засчитывается в прогресс |
| ❌ **Отклонён** | TestLab — не пройден порог, либо эксперт отклонил (уровень 4) | Отображается с комментарием (для уровня 4 — обязателен всегда, см. ниже) |

### Флоу подтверждения:
1. Сотрудник указывает уровень навыка
2. Для уровней 1–3 — статус **Ожидает подтверждения**, результат приходит автоматически от BILIM/TestLab, без участия сотрудника или ответственного лица
3. Для уровня 4 — сотрудник нажимает «Подтвердить навык», заявка уходит эксперту/руководителю в BPM
4. Эксперт нажимает «Подтвердить» или «Отклонить» — **комментарий обязателен в обоих случаях** (раньше был обязателен только при отклонении; расширено по итогам встречи 23.07.2026)
5. Сотрудник получает уведомление о результате с комментарием эксперта в любом случае

## 3. Аттестация (интеграция с БПМ)

### Флоу:
1. Сотрудник в разделе «Аттестация» нажимает **«Запросить аттестацию»**
   - Доступно только если: % выполнения плана ≥ **85%** и ключевые навыки подтверждены
2. Система находит HR-партнёра, закреплённого за подразделением сотрудника (данные из БПМ)
3. В БПМ на HR падает задача: _«Сотрудник [ФИО], должность [X], заявил готовность к аттестации на [целевую должность]»_
4. HR в БПМ:
   - Добавляет членов комиссии (выбирает людей)
   - Проверяет доступность слотов у всех участников
   - Бронирует аудиторию
   - Нажимает **«Исполнить»**
5. Сотрудник получает уведомление: дата, время, место аттестации
6. На главной странице сотрудника появляется виджет «Ближайшая аттестация»
7. После аттестации HR фиксирует результат в БПМ → данные приходят в BI DAMU → статус плана обновляется

### Данные из БПМ:
- Оргструктура (кто чей руководитель, кто HR подразделения)
- Результаты аттестаций

## 4. Уведомления

Все уведомления приходят в колокольчик в шапке и (опционально) на email/Telegram.

| Событие | Кому |
|---------|------|
| HR обновил матрицу навыков для должности | Всем сотрудникам этой должности |
| Запрос на подтверждение навыка | Руководителю / эксперту / HR |
| Навык подтверждён / отклонён | Сотруднику |
| Аттестация назначена (дата/место) | Сотруднику |
| Сотрудник запросил аттестацию | HR-партнёру |
| Срок карьерного плана истекает (за 30 дней) | Сотруднику и руководителю |

## 5. Карьерная карта — поиск и группировка должностей

### Поле «До» (выбор целевой должности):
- Должности сгруппированы по подразделениям/функциям
- Пример раскрытия: **Финансы** → Экономист / Финансовый менеджер / Финансовый директор
- MVP: только ИТР-состав (Foreman + Site Manager и смежные треки)

### Поиск на карте:
- Отдельное поле — поиск по конкретному названию должности
- Находит и подсвечивает карточку на карте
- Не связан с группировкой в «До»

## 3.1 Подтверждение проекта (опыта работы) — новая сущность

> Зафиксировано по итогам встречи с заказчиком 23.07.2026. Полное описание флоу — [BPM_INTEGRATION.md, раздел 4](BPM_INTEGRATION.md#4-подтверждение-проекта-опыта-работы).

Контекст от HRD: разброс по опыту сотрудников большой — ~40% Foreman'ов (молодые, 22–25 лет) успели сдать всего 1–2 объекта, ~10% — со стажем от 5 лет. Компании нужно реально собрать и подтвердить вклад каждого сотрудника в проекты.

- Сотрудник добавляет проект в «Профиль опыта» вручную, как и раньше (форма не меняется)
- Каждый добавленный вручную проект уходит **отдельной задачей HR-партнёру** в UnityBPM — HR подтверждает участие сотрудника в проекте офлайн (кадровые документы, созвоны с причастными), **по одному проекту за раз**, не пакетно
- Статус проекта: `pending` → `confirmed` / `rejected`
- Как только заработает интеграция с SuperApp — **новые** проекты будут приходить оттуда автоматически, без участия HR и без прохождения через этот BPM-флоу; ручное подтверждение HR останется только для проектов, введённых сотрудником до этой интеграции
